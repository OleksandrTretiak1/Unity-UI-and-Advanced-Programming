# Advanced C# Scripting & UI Lab — Третяк Олександр

Цей репозиторій містить виконання лабораторної роботи з поглибленого вивчення C# та розробки інтерфейсів користувача в Unity.

## Користувацький інтерфейс (UI)
Робота над UI була виконана поетапно згідно з модулем "Get started with user interfaces". Основна увага була приділена створенню меню та елементів керування, які коректно відображаються на різних екранах.
**Ключові етапи реалізації:**
* **Основи Canvas:** Ознайомлення з базовими принципами роботи системи UI в Unity.
* **Текстові елементи (Title):** Створення та налаштування заголовків для сцен.
* **Адаптивність (Anchors):** Налаштування якорів та точок повороту (Pivot) для забезпечення правильного масштабування елементів при зміні розміру екрана.
* **Фонові зображення:** Створення візуального фону для меню з використанням UI Images.
* **Кнопки (Button Functionality):** Додавання стандартних кнопок та налаштування їхньої базової функціональності (взаємодія при натисканні).
* **Елементи керування (Toggles & Sliders):** Реалізація перемикачів та повзунків для створення меню налаштувань.
* **Worldspace UI (Exercise):** Виконання практичного завдання зі створення інтерфейсу, що розміщений безпосередньо у тривимірному ігровому просторі.

## Поглиблений C# (Advanced Scripting)
Практична реалізація концепцій програмування представлена у вигляді **серії окремих навчальних скриптів**, що знаходяться за шляхом:
`Assets/CreativeCore_UI/Scripts/AdvancedCSharpLab/`

Кожен файл демонструє конкретний архітектурний підхід ООП:

### Архітектура та Організація
* **Namespaces:** Логічне групування коду та уникнення конфліктів імен.
* **Statics:** Створення спільних змінних та методів, доступних через назву класу без створення об'єктів.
* **Properties:** Використання інкапсуляції для безпечного керування даними через `get` та `set`.

### Спадкування та Поліморфізм
* **Inheritance:** Побудова ієрархії класів з передачею параметрів у батьківські конструктори через `base`.
* **Polymorphism:** Здатність обробляти об'єкти різних типів (дитячих класів) через посилання на їхній загальний тип (батьківський клас).
* **Overriding:** Зміна поведінки методів у дочірніх класах за допомогою механізму `virtual` та `override`.
* **Member Hiding:** Приховування методів базового класу за допомогою ключового слова `new`.

### Гнучкість та Розширення
* **Interfaces:** Створення спільних контрактів (наприклад, `IDamageable`) для взаємодії між незалежними системами.
* **Method Overloading:** Реалізація декількох методів з однаковою назвою, але різними параметрами.
* **Generics:** Написання універсального коду за допомогою узагальненого типу `<T>`.
* **Extension Methods:** Додавання нових функцій до вбудованих компонентів Unity (наприклад, до `Transform`).
